picoRTOS_mutex_init

void picoRTOS_mutex_init(struct picoRTOS_mutex *mutex)

Initialises a mutex

Mutexes are re-entrant on picoRTOS, if you don’t need this feature, prefer futexes

Parameters

mutexA pointer to the mutex to initialize

Remarks

Another way to initialise a mutex at startup is this one:

struct picoRTOS_mutex mutex = PICORTOS_MUTEX_INITIALIZER;

picoRTOS_mutex_lock

void picoRTOS_mutex_lock(struct picoRTOS_mutex *mutex)

Acquires a mutex

Parameters

mutexThe mutex you want to acquire

Remarks

The fuction will make CONFIG_DEADLOCK_COUNT attempts to acquire the mutex, if it fails, picoRTOS will throw a debug exception and declare deadlock

picoRTOS_mutex_unlock

void picoRTOS_mutex_unlock(struct picoRTOS_mutex *mutex)

Releases a mutex

Parameters

mutexThe mutex you want to release

Remarks

picoRTOS will throw a debug exception if :

  • the mutex has not been acquired
  • the mutex has already been released
  • you are not the owner
